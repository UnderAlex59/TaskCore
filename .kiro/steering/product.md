# Продуктовый контекст

## Назначение

Репозиторий реализует MVP интеллектуальной платформы управления задачами. Система помогает команде фиксировать требования, проверять качество описания задач, обсуждать задачу в контекстном чате, сохранять предложения изменений и переиспользовать проектную память через RAG.

Документация в `.kiro` описывает только текущее реализованное состояние. Она предназначена для разработчиков и ИИ-агентов, которым нужно безопасно менять систему без потери доменных инвариантов.

## Акторы

| Роль | Назначение в системе |
| --- | --- |
| `ADMIN` | Управляет пользователями, LLM runtime, мониторингом, evaluation-контурами, глобальными тегами и проектными настройками валидации. |
| `MANAGER` | Управляет проектом, участниками и рабочим процессом задачи в пределах проектных прав. |
| `ANALYST` | Создает и уточняет задачи, запускает валидацию, участвует в согласовании требований и чате задачи. |
| `DEVELOPER` | Получает утвержденные задачи, переводит их в разработку и готовность к тестированию. |
| `TESTER` | Получает задачу на тестирование и завершает workflow после проверки. |

## Основные сценарии

- Регистрация и вход пользователя через JWT access token и refresh cookie.
- Создание проекта, добавление участников и настройка проектных правил.
- Создание задачи с Markdown-описанием, тегами и вложениями.
- Автоматическая валидация требования через LangGraph: базовые правила, кастомные правила проекта, контекстные вопросы.
- Согласование задачи аналитиком/ревьюером и назначение разработчика и тестировщика.
- Прохождение задачи по статусам `draft -> validating -> awaiting_approval -> ready_for_dev -> in_progress -> ready_for_testing -> testing -> done`, с ответвлением `needs_rework`.
- Контекстный чат задачи с LLM-маршрутизацией к QA Agent, ChangeTracker Agent или Manager Agent.
- Индексация задачи, вложений, валидации и предложений изменений в Qdrant.
- Уведомления в интерфейсе, WebSocket-обновления и optional Telegram-доставка.
- Админский контур для LLM providers, prompt configs, graph monitoring, Qdrant diagnostics и evaluation-наборов.

## Границы текущего продукта

- Система является MVP и не содержит биллинга, внешнего SSO, multi-tenant изоляции за пределами проектного membership и production-grade фонового worker.
- Evaluation-контуры доступны через админские API/UI и запускаются как background tasks FastAPI.
- Frontend использует single-page React-приложение; production-like сборка ожидает API под `/api`.
- RAG зависит от настроенного embedding provider и Qdrant; LLM-возможности зависят от runtime provider configs и env-настроек.
